## Background
### Overview
A production weekly surveillance pipeline for West Nile Virus (WNV) and St. Louis Encephalitis Virus (SLEV) in Culex mosquitoes across Boulder County / Fort Collins, Colorado. Processes qPCR, trap count, and pool test data each week and outputs a formatted Excel report, CSVs, and plots for public health epidemiologists.

### Details
Calculating abundance: 
- defined as the average number of mosquitoes per trap per night
- light traps are used exclusively for abundance and needs to be calculated by species. 
- Company that collects traps doesn't explicitly state when tarsalis or pipiens are 0 in a given trap. This must be imputed.
- malfunctioning traps are NA. They were never set and should be dropped from the calculation
- each week we expect the following number of traps to be accounted for:

| Zone        | G  | L  | Grand Total |
|-------------|----|----|-------------|
| BC          |    | 5  | 5           |
| BE          |    | 10 | 10          |
| LV          |    | 6  | 6           |
| NE          | 4  | 10 | 14          |
| NW          | 2  | 9  | 11          |
| SE          | 3  | 15 | 18          |
| SW          | 1  | 9  | 10          |
| **Total**   | **10** | **64** | **74** |

note: for BC could be more.

Calculating PIR:
- PIR defined as the pooled infection rate within a trap
- calculated using CDC package using the firth method
- PIR includes gravid traps in its calculations
- PIR roughly assumes 1 infected mosquito per trap. #positive pools/#mosquitoes

Calculating VI:
- defined as the number of infected mosquitoes per trap per night

## Commands

```bash
# 1. Generate config (run once per week — sets all pipeline parameters)
bash config/run_config.sh --week <N> --year <YYYY> --download T --update F --push F

# 2. Render the main pipeline
quarto render wnv-s_weekly_report_pipeline_v2.qmd

# Run unit tests
Rscript tests/run_tests.R
```

**`--download T`** pulls fresh inputs from Google Drive. Use `--download F` to skip (offline work or re-running on existing files). **`--update F`** prevents writing back to the master Google Sheet — keep `F` until the run is validated.

## Non-Standard Package

`PooledInfRate` is not on CRAN. Install with:

```r
devtools::install_github("CDCgov/PooledInfRate")
```

Without it, `calc_pir()` fails at runtime with no clear error message.

## Config Must Run Before the Pipeline

`config/run_config.sh` saves a timestamped `.RData` snapshot to `config/config_weekly_settings/`. The QMD loads the most recent snapshot at the top. If config hasn't been re-run for the current week, the pipeline silently uses last week's parameters — always check the timestamp.

## Google Sheets Auth

`googlesheets4` and `googledrive` use a cached OAuth token in `~/.config/gargle/`. Authenticate once interactively in RStudio; subsequent runs use the cache. Never modify the token file directly. To re-authenticate: `googledrive::drive_auth()`.

## Testing

Unit tests live in `tests/testthat/` (testthat framework), covering `calc_abund`, `calc_pir`, and `calc_vi`. Run with `Rscript tests/run_tests.R`. Fixtures use synthetic data generated by `tests/fixtures/generate_fixtures.R`. When adding or modifying functions in `utils/`, add or update the corresponding test.

## Key Gotchas

- **Zone factor levels.** All zone columns must be coerced to the `zone_lvls` factor defined in `config/config_weekly.R`. Skipping `wnv_s_clean()` causes silent grouping errors in `calc_pir()` and `calc_vi()`.
- **Missing trap entries.** `NA` means the trap was not run; `0` means it ran with zero catch. These are treated differently in abundance calculations — do not coerce one to the other.
- **PCR edge cases.** Inconclusive PCR results require manual QC in Thermofisher Quantstudio software before running the pipeline.
- **GSheet keys are hardcoded in config.** Sheet IDs live in `config/config_weekly.R` lines 178–189. Don't move them to YAML without discussing the change first.
- **Data entry errors.** The most common source of unexpected results is upstream data entry in the datasheet. Check the raw input before assuming a function bug.
