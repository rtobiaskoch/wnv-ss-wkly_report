## Notes / Sharp Edges
- **Stale-config trap:** the qmd loads the *newest* snapshot via `read_latest("config/config_weekly_settings")`. If a 2026 w23 snapshot isn't there, it silently renders the last week (w37/2025) and writes to that output dir ‚Äî looks like "0 output files" for the week you expected.
- **`download T` re-pulls the key:** with download on, `gsheet_pull_prompt(fn_key_rename,...)` overwrites the local key with the GSheet's 1-space variant, re-triggering the trap_id drop. Use `--download F` for local validation.
- **Frozen-baseline gate:** `wnv_s_clean` only computes `week` when the column is absent (`!"week" %in% names(df)`), so a stored master keeps its weeks on read-back ‚Äî by design, to hold the historical line stable mid-season. Means: to get corrected weeks into the report you must swap in the corrected master, not rely on recompute.
- **Sandbox limits:** cannot build/install R packages or write the user R library ‚Äî the user runs `devtools::document()`/`remotes::install_local()` themselves via `! ‚Ä¶`. After install, **restart R** to load the new package.
- **zsh `!` mangling:** write diagnostic R as files and `Rscript file.R`; avoid `Rscript -e '...!...'` and heredocs.
- Active code lives in `utils/`; ignore `utils/dev|archive|graveyard` and `0_R/` (one stale `isoweek` ref in `utils/dev/` is inactive). qmd:18 has a stale prose TODO "Change Boulder week to epiweek" ‚Äî outdated, harmless.
